Enhanced Search — Privacy Policy

Last updated:

1. Data collection

The extension performs all search, highlighting, and UI operations locally in the browser. No page content, keystrokes, or search queries are transmitted off-device unless you explicitly use an export feature to save a file locally.

2. Remote code and optional downloads

Optional remote assets (for example, language packs or an optional Web Worker bundle) are downloaded only after explicit user opt-in. Remote resources are fetched over HTTPS, verified with Subresource Integrity (SRI) where possible, and executed without dynamic eval. Users can disable remote resources in settings; the extension falls back to a fully local implementation.

3. Permissions and usage

  • storage: used to persist user preferences, overlay position, and local search history.
  • activeTab / scripting: used only to inject the overlay and perform in-page operations when the user activates the extension.
  • contextMenus: used only to add an optional right-click entry to open the search overlay.
  • Host access: minimal and only used when the user enables the extension for a site; no page content is uploaded.

4. Retention & deletion

All persisted data is stored in the browser (Chrome storage.local) and can be cleared from the extension’s options page or via the browser’s extension settings.

5. Security & contact

No user data is shared with third parties. For security concerns or questions, contact: privacy@yourdomain.example (replace with your contact address).
